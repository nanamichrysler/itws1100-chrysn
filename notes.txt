That's a common issue! Your footer isn't at the very bottom of the page because of how your PHP code structure interacts with your CSS's sticky footer technique.

In this specific file, the PHP script handles form submission and conditionally outputs content (either an error message or the "Thank you!" message).

🧩 The Problem: Premature </body>
The core issue is that your PHP script, upon successful submission (or error), executes a return statement or finishes its display, and then the HTML structure ends.

When a comment is submitted, your PHP code runs, prints the "Thank you!" message inside the <div class="commentAccepted"> block:

PHP

if ($stmt->execute()) {
    echo '<div class="commentAccepted">';
    echo "<h4>Thank you! <br> Your comment has been submitted.</h4><br>";
    echo '<a href="../../index.php" class="button cta">Return to Home</a>';
    echo '</div>';
} // ... rest of the file
The PHP script stops here (or just continues to the end).

The rest of your HTML file, which contains your contact icons and the included footer, is then rendered:

HTML

<div class="contact-icons">
</div>
<?php include('footer.php'); ?>
</body>
</html>
If the content generated by the PHP script (the "Thank you!" message) is very short, the total content height is less than the viewport height.

The CSS Sticky Footer technique you are using relies on the main content taking up the space between the banner and the footer:

CSS

body {
  /* ... other styles ... */
  min-height: 100vh; /* 100% of the viewport height */
  position: relative;
}
/* The footer must be positioned absolutely or fixed at the bottom */
/* Your included footer.php likely contains CSS/HTML to make it stick */
When the content is short, the footer snaps to the bottom of that short content, which is then forced to the bottom of the viewport by the min-height: 100vh on the body. However, in this PHP file, the footer is not part of the main content area; it comes after the main content.

🛠️ The Solution: Restructure the PHP/HTML
You need to wrap the dynamic PHP content in a main content container, and ensure that your footer HTML is consistently placed just before the closing </body> tag and contains the necessary CSS for a sticky footer.

Since you've shown the HTML for contact-icons, I'll assume that section and your included footer.php are intended to be the actual footer area.
